## 🌱 Ways to Contribute

- ➕ Add a new resource to an existing category
- 🔗 Fix a broken or outdated link
- ✍️ Improve existing descriptions or documentation
- 📂 Suggest a new category
- 🐛 Report bugs, typos, or formatting issues
- 💡 Suggest new features for the project itself

---

## ✅ Before You Start

- Search existing issues and pull requests to avoid duplicates
- For large changes (new categories, structural changes), open an issue first to discuss
- Small fixes (typos, broken links) can go straight to a pull request

---

## 📥 Adding a New Resource

1. **Fork** this repository and clone your fork
2. **Find the right category** file inside `categories/` (or propose a new one if none fits)
3. **Copy the format** from [`templates/RESOURCE_TEMPLATE.md`](templates/RESOURCE_TEMPLATE.md)
4. **Fill in every field** — incomplete entries will be asked for revision
5. **Keep alphabetical order** within the category file
6. **Commit, push, and open a pull request**

### Example Entry

**[Visual Studio Code](https://code.visualstudio.com/)**

A free, open-source code editor developed by Microsoft, supporting nearly every language and framework through extensions.

- **GitHub Repository:** [microsoft/vscode](https://github.com/microsoft/vscode)
- **Features:** IntelliSense, built-in Git, integrated terminal, huge extension marketplace
- **Advantages:** Lightweight, highly customizable, massive community and ecosystem
- **Disadvantages:** Can slow down with too many extensions installed
- **Free/Paid:** Free
- **Alternatives:** Sublime Text, JetBrains IDEs, Neovim
- **Tags:** `editor` `ide` `free` `cross-platform`

## 🎨 Style Guide

- Keep descriptions to 1–3 sentences — clear over clever
- Write in present tense, active voice
- Avoid marketing language ("revolutionary", "best-in-class") — stay factual
- Test every link before submitting
- Use the exact field names and order from the template

---

## 📝 Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` a new resource or category added
- `fix:` broken link, typo, or factual correction
- `docs:` documentation-only changes
- `chore:` maintenance, tooling, or structural changes

Example: `feat: add Visual Studio Code to Developer Tools`

---

## 🔃 Pull Request Process

1. Fill out the pull request template completely
2. Ensure all links in your change work
3. One resource or category per pull request keeps reviews fast
4. A maintainer will review and may request changes
5. Once approved, it's merged — you're now a Tool Verse contributor! 🎉
